Call Transfer Flow

April 2026
When the AI agent determines that a customer needs to speak with a human agent, it initiates a call transfer. This document explains what happens during that transfer — from both the customer’s and the callcenter agent’s perspective.

1. Overview

The transfer process is designed to ensure the human agent has full context before speaking with the customer — no cold transfers. The customer hears hold music while a support ticket is created and the agent is briefed with an AI-generated summary of the conversation.

END-TO-END TRANSFER FLOW —————————————————————————————————————— AI agent decides customer needs human support | v Customer hears: "I’ll transfer you now..." | v Customer is placed on hold with music (~15–30 seconds) | | | In parallel: | | • Support ticket is created | | • Callcenter number is dialed | | • IVR navigated automatically | | (if callcenter has a menu) | | | v v Agent’s phone rings Ticket ready | v Agent picks up → hears: "Press 1 to hear the summary of this call" | |—— Presses 1 → AI summary is read aloud (~15–30 seconds) |—— Presses 2 → Ticket number repeated |—— Presses 3 → Full summary repeated | v Agent presses 1 (again) → Connected to customer | v Customer + Agent in live conversation
Typical total transfer time: 30–60 seconds from the moment the AI decides to transfer until the customer is speaking with a human agent. This includes ticket creation, agent dialing, and the whisper summary.

2. What the Customer Experiences

1
Transfer announcement

The AI agent tells the customer it will connect them with a human agent. The exact wording is customisable per company and language.

2
Hold music

The customer is placed in a waiting room with hold music. The music is customisable (your own audio file or our default). During this time, the support ticket is being created and the callcenter agent is being dialed. If your callcenter has an IVR menu, we navigate it automatically on your behalf (configurable DTMF sequences).

3
Connected to agent

Once the callcenter agent has heard the summary and is ready, the customer is connected. From the customer’s perspective, the hold music stops and they’re speaking with the agent.

If no agent is available: If the callcenter doesn’t answer within the configured timeout, the customer hears a message (e.g. “Unfortunately no agent is available at the moment. Please try again later.”). This message is customisable. Failover to a secondary callcenter number can be configured.

3. What the Callcenter Agent Experiences

1
Phone rings

The agent’s phone rings like any other incoming call. The caller ID shows the Lemonflow transfer number (configurable).

2
Agent picks up → hears menu prompt

Instead of being connected to the customer immediately, the agent hears a brief prompt: “Press 1 to hear the summary of this call.”

3
Agent presses 1 → hears AI summary

A text-to-speech voice reads an AI-generated summary of the conversation. This includes:

  • The company name (e.g. “Incoming call from [Company] customer support”)
  • A concise summary of the customer’s issue and what the AI already tried
  • The support ticket number (if a ticketing system is connected)
4
Agent chooses next action

After hearing the summary, the agent can:

KeyAction
Press 1Connect to the customer
Press 2Hear the ticket number again
Press 3Hear the full summary again
5
Connected to customer

The agent is now live with the customer. The agent has full context from the AI summary and (if applicable) the support ticket is already open in their system.

Auto-play mode: For callcenters that prefer it, the summary can be played automatically when the agent picks up — no “press 1” step. The agent simply listens to the summary, then presses 1 to connect. This is a configuration option.

4. Transfer Methods

We support two methods for dialing the callcenter. Both provide the same agent experience (whisper summary, menu, conference bridge). The choice depends on your telephony setup.

MethodHow It WorksBest For
PSTN We dial the callcenter’s regular phone number. Works out of the box — no configuration needed on your side beyond providing the number. Quick setup, no SIP infrastructure required
SIP We connect directly to the callcenter’s SBC/PBX via SIP. Lower latency, lower cost, and enables passing metadata (e.g. ticket ID) via SIP headers for automatic agent screen-pop. Callcenters with SIP infrastructure who want automatic ticket pop-up
SIP screen-pop: With SIP transfers, we can include the support ticket ID in the User-to-User SIP header. If your CRM/ACD supports UUI, the ticket automatically opens on the agent’s screen when they answer — no manual lookup needed.

5. What’s Configurable

The transfer flow is highly configurable. Below is a selection of the most commonly used settings — all can be tailored per company and per language. Contact tech@lemonflow.ai for the full list of options.

Routing & Failover

SettingDescription
Callcenter number(s) The phone number(s) or SIP URI(s) we dial. Multiple destinations supported (e.g. per language, per queue).
Time-based routing Automatically route to different numbers based on time of day or day of week (e.g. daytime team vs. after-hours overflow, weekday vs. weekend).
Failover If the primary callcenter doesn’t answer within the timeout, automatically try a secondary number.
Agent wait timeout How long the customer waits before the “no agent available” message plays. Configurable per destination.

Customer Experience

SettingDescription
Hold music Custom audio file for the customer while waiting. Can be your own branded music or our default.
Transfer announcement The message the AI says before transferring (e.g. “I’ll connect you with a colleague now”). Customisable wording per language.
No-answer message What the customer hears if no agent is available (e.g. “Please try again later” or “We’ll call you back”). Customisable per language.

Agent Experience

SettingDescription
Summary mode Auto-play the summary immediately when the agent answers, or prompt “press 1” first.
Whisper voice & language The TTS voice used for the agent whisper. Matched to the callcenter agent’s language.
Summary content What’s included in the whisper: company name, conversation summary, ticket number, custom prefix/suffix text.
IVR navigation If your callcenter has an IVR menu before reaching an agent, we can automatically navigate it (configurable DTMF sequences and pauses).
Speech recognition Allow the agent to say “one” instead of pressing 1 on the keypad.

Recording & Monitoring

SettingDescription
Call recording Record the conversation after transfer. Dual-channel supported (customer + agent on separate audio tracks).
Transfer outcome logging Track whether the transfer was successful, agent didn’t answer, or customer hung up during hold.
This is not an exhaustive list. The transfer flow supports many additional configuration options. Contact tech@lemonflow.ai to discuss your specific requirements.
