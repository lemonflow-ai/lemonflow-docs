Voice Inbound — SIP Trunking

April 2026
This document covers how to route inbound calls to Lemonflow via SIP. It is intended for your telco provider or telephony team. Lemonflow supports both PSTN and SIP for inbound calls — PSTN works out of the box with no partner configuration, SIP is recommended for lower latency and lower per-minute cost. This document covers the SIP setup.

1. Lemonflow SIP Endpoint

ParameterValueNotes
SIP Domain lemonflow-inbound-ip.sip.twilio.com IP ACL authentication (recommended)
SIP Domain (alt) lemonflow-inbound.sip.twilio.com Credential (digest) authentication
SIP URI format sip:{DID}@lemonflow-inbound-ip.sip.twilio.com DID in E.164 with leading +
Signaling IPs 35.156.191.128/30 Frankfurt (EU — primary)
Signaling IPs (failover) 54.171.127.192/30 Ireland (EU — failover)
Media IP range 168.86.128.0/18 RTP media
Media port range 10000–60000 UDP
SIP port 5060 (UDP/TCP) / 5061 (TLS) Both supported
Codec G.711 (PCMU & PCMA) Both u-law and a-law
DTMF RFC 2833

2. Authentication

We support two methods. Both can be active simultaneously on different domains.

MethodSIP DomainHow It WorksBest For
IP ACL (recommended) lemonflow-inbound-ip.sip.twilio.com We whitelist your telco’s signaling gateway IPs. Only traffic from those IPs is accepted. Telco providers with static gateway IPs
SIP Digest lemonflow-inbound.sip.twilio.com We create a username/password credential pair. The telco authenticates via standard SIP Digest (RFC 3261). Providers without static IPs

3. What We Need from Your Telco

ItemDescription
DIDs (phone numbers) The list of phone numbers being routed to us. This is how we determine which AI agent to connect the caller to — one DID per language or region.
Signaling gateway IPs For IP ACL authentication — the gateway IP(s) or CIDR range from which SIP INVITEs will originate. (If using Digest auth instead, we provide the credentials.)
SIP trunk configuration Route the DIDs above to lemonflow-inbound-ip.sip.twilio.com (IP ACL) or lemonflow-inbound.sip.twilio.com (Digest).
Firewall rules Allow outbound SIP signaling to 35.156.191.128/30 (port 5060/5061) and media to 168.86.128.0/18 (ports 10000–60000).
Caller ID preservation Preserve the original caller’s number in the From header (don’t send anonymous).
DID in To header Send the dialed number in E.164 format in the To / Request-URI. This is how we route to the correct AI agent.

4. Call Flow

INBOUND SIP CALL FLOW Caller dials DID (e.g. the support number for Germany) | v Telco gateway | | SIP INVITE | From: caller number | To: {DID} in E.164 | v Lemonflow SIP endpoint —— IP ACL check (or digest auth) | | Looks up DID → finds route | e.g. +4930... → AI agent (DE) | v AI agent handles call in the correct language

5. Security

LayerProtocolDetails
Signaling encryption TLS 1.2+ SIP over TLS on port 5061. Encrypts all SIP signaling.
Media encryption SRTP Supported when both sides negotiate it.
Authentication IP ACL or SIP Digest IP ACL: only whitelisted IPs accepted. Digest: RFC 3261 challenge–response.
Infrastructure EU region All traffic processed via EU infrastructure (Frankfurt primary, Ireland failover). Underlying infrastructure is SOC 2 Type II and ISO 27001 certified.
